Event Loop

"Macrotasks և Microtasks: Ինչպե՞ս է աշխատում JavaScript-ի շարժիչը:"

01

Ի՞նչ է Event Loop-ը

JavaScript-ի գործարկման հիմքում ընկած է Event Loop (Իրադարձությունների ցիկլ) գաղափարը: Այն շատ պարզ է՝ կա անվերջ ցիկլ, որտեղ JavaScript-ի շարժիչը սպասում է առաջադրանքների (tasks), կատարում է դրանք, իսկ հետո "քնում է"՝ սպասելով նոր առաջադրանքների:

UI Rendering: Կարևոր է իմանալ, որ բրաուզերը երբեք չի նկարում (render) էկրանը, քանի դեռ շարժիչը կատարում է որևէ առաջադրանք: Փոփոխությունները արտացոլվում են միայն այն ժամանակ, երբ առաջադրանքն ավարտվում է:
02

Տեսակները

Macrotasks (Մակրոթասքեր)

Սրանք հիմնական առաջադրանքներն են, որոնք գալիս են բրաուզերից կամ արտաքին միջավայրից.

  • script (ամբողջական ֆայլի բեռնում և գործարկում)
  • Օգտատիրոջ գործողություններ (mousemove, click)
  • Ժամանակացույցեր (setTimeout, setInterval)

Microtasks (Միկրոթասքեր)

Միկրոթասքերը ստեղծվում են հենց մեր կոդի կողմից: Դրանք ավելի բարձր առաջնահերթություն ունեն, քան մակրոթասքերը.

  • Promise-ներ (.then, .catch, .finally)
  • await գործողություններ
  • queueMicrotask(func)

04

Գործնական Օրինակ

Ի՞նչ հերթականությամբ կտպվեն թվերը console-ում:

console.log(1);

setTimeout(() => console.log(2));

Promise.resolve().then(() => console.log(3));

Promise.resolve().then(() => setTimeout(() => console.log(4)));

Promise.resolve().then(() => console.log(5));

setTimeout(() => console.log(6));

console.log(7);

Օգտակար Կիրառություններ

1. CPU ծանրաբեռնող խնդիրների բաժանում

Եթե ունեք շատ ծանր ցիկլ (օրինակ՝ գույների փոփոխություն մեծ տեքստում կամ 1 միլիարդ հաշվարկ), բրաուզերը կկախի ("hang"), և կոճակները չեն աշխատի: Կոդը կարելի է բաժանել մասերի և յուրաքանչյուր մասը դնել setTimeout(func)-ի մեջ: Այդպիսով, Event Loop-ը «շունչ կքաշի» և կկարողանա այլ բաներ անել (օրինակ՝ արձագանքել click-երին) կտորների արանքում:
