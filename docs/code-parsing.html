JIT Optimization (V8 Pipeline)

Շարժիչը չի սպասում ամբողջ կոդի թարգմանությանը։ Այն օգտագործում է Ignition (Interpreter)՝ բայթ-կոդի արագ ստեղծման համար, և TurboFan (Optimizing Compiler), որը վերլուծում է "Hot" ֆունկցիաները և դրանք վերածում օպտիմիզացված մեքենայական կոդի։

Hidden Classes (Shapes)

V8-ը ստեղծում է Hidden Classes (հայտնի նաև որպես Shapes): Սա թույլ է տալիս կիրառել Inline Caching (IC) տեխնոլոգիան, որը ապահովում է Property Access-ի արագությունը՝ հիշողության մեջ տվյալների ֆիքսված տեղակայման (Memory Offset) շնորհիվ:

Garbage Collector (GC)

JS-ը կիրառում է Automatic Memory Management: Հիմնական Garbage Collector-ը աշխատում է "Mark-and-Sweep" ալգորիթմով։ Այն ազատում է հիշողությունը (Deallocation) այն օբյեկտներից, որոնք այլևս Unreachable են՝ կանխելով Memory Leaks-ը։
