Garbage Collector

"Garbage Collection. Հիշողության կառավարման արվեստը JavaScript-ում"

Memory

Ծրագրի կողմից օգտագործվող տարածքը: JS-ը ավտոմատ է կառավարում հիշողության հատկացումն ու ազատումը։

Root

Ներկայիս ակտիվ տվյալները (Global objects, Stack), որտեղից GC-ն սկսում է հասանելիության ստուգումը։

Stack

Լոկալ փոփոխականներ և ֆունկցիաների կանչեր (Execution Context):

Heap

Օբյեկտներ, զանգվածներ և ֆունկցիաներ: Այստեղ տվյալները դինամիկ են։

01

Memory Management in JS

Ի տարբերություն ցածր մակարդակի լեզուների (ինչպիսին է C-ն), JS-ն ունի ավտոմատ հիշողության կառավարում։

Garbage Collector-ը (GC) անտեսանելի գործընթաց է, որը պարբերաբար ստուգում է հիշողությունը և հեռացնում այն ամենը, ինչն այլևս «հասանելի» չէ ծրագրին։

Stack vs Heap

Stack (Ստեկ)

Պահպանում է Value Types (number, string, boolean) և ֆունկցիաների կանչերը։ Աշխատում է շատ արագ, քանի որ ունի ֆիքսված կառուցվածք (LIFO)։

Heap (Դիզակ)

Պահպանում է Reference Types (objects, arrays)։ Այստեղ տվյալների չափը կանխատեսելի չէ, ուստի կառավարումը պահանջում է ավելի շատ ռեսուրսներ։

02

Reachability (Հասանելիություն)

GC-ի հիմնական սկզբունքը հասանելիությունն է։ Տվյալները մնում են հիշողության մեջ, եթե դրանք կարելի է գտնել՝ սկսելով Roots-ից։

  • Գլոբալ օբյեկտներ (window կամ global)
  • Կանչերի ստեկում գտնվող լոկալ փոփոխականներ
  • Ակտիվ Closure-ների մեջ պահվող փոփոխականներ
03

Mark-and-Sweep Algorithm

Սա ժամանակակից JS շարժիչների (V8, SpiderMonkey) հիմնական ալգորիթմն է։

1. Mark (Նշել)

Գլոբալ աղբյուրներից (roots) սկսելով՝ GC-ն «ներկում» է բոլոր հասանելի օբյեկտներին: Եթե մի օբյեկտ հասանելի չէ, նշիչը չի դնի։

2. Sweep (Մաքրել)

Բոլոր այն օբյեկտները, որոնք մնացել են նշված, համարվում են հասանելի և չեն ջնջվի։

04

Generational Garbage Collection

Ժամանակակից GC-ները (հատկապես V8) օգտագործում են սերունդային մոտեցում, քանի որ օբյեկտների մեծ մասը «մահանում է երիտասարդ»։

Young Generation

Այստեղ են ստեղծվում նոր օբյեկտների մեծ մասը։ GC-ն աշխատում է շատ հաճախ և արագ (Minor GC / Scavenger)։

Old Generation

Օբյեկտները, որոնք գոյատևել են մի քանի Minor GC, տեղափոխվում են այստեղ։ Մաքրումը տեղի է ունենում ավելի հազվադեպ (Major GC)։

05

Common Memory Leaks

Չնայած ավտոմատ հիշողության կառավարմանը, JS-ում հնարավոր է «հիշողության արտահոսք» (memory leak)։

Հիշողության արտահոսք տեղի է ունենում, երբ մենք պատահաբար պահում ենք հղումներ այն օբյեկտների վրա, որոնք մեզ այլևս պետք չեն։

  • Global Variables: Պատահական ստեղծված գլոբալ փոփոխականները երբեք չեն ջնջվում։
  • Forgotten Timers: Չդադարեցված setInterval-ները շարունակում են պահել հղումներ։
  • Detached DOM nodes: HTML-ից հեռացված էլեմենտները մնում են հիշողության մեջ, եթե JS-ում հղում կա։
  • Uncleared Event Listeners: addEventListener առանց removeEventListener-ի։
  • Closure-based Leaks: Ներքին ֆունկցիաները կարող են պահել արտաքին ֆունկցիաների փոփոխականների հղումները, նույնիսկ այն բանից հետո, երբ արտաքին ֆունկցիան ավարտվել է։
